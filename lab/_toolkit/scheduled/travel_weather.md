你的任务是查询一个固定日期区间的旅行天气预报，覆盖出发城市和每一个目的地。
所有天气从 Open-Meteo API 获取数据，不采信任何其他渠道的数据。天气 API 的调用方法见 `agent-workspace/.memory/entities/open_meteo_api.md`。

## 变量定义

出发城市：杭州

| 目的地 | 日期区间        |
| --- | ----------- |
| 景洪市 | 3月25日-3月30日 |

## 执行步骤

1. 调用地理编码 API 获取每个城市的经纬度坐标。
2. 查询{日期区间}{目的地城市}每一天的天气。
3. 从中提取出气温和降水概率数据，列出来。
4. 查询{日期区间}里最早1天和最晚1天{出发城市}的气温和降水概率，写在开头和末尾。
5. 如果查询日期超过预报范围，可以写"暂无预报"。
6. 使用utils/api_telegram.py，把结果发送出去。

整个仓库里你只能使用api_telegram.py，其他全靠你自己手动完成。
执行过程不需要对代码库做任何改动，也不需要提交任何PR。如果产生了任何临时文件，都应该在成功发送到Telegram之后删除。

**03.25 杭州出发（12-19℃，降水概率 10%）**

**景洪市**
- 03.25（19-28℃，降水概率 3%）
- 03.26（19-28℃，降水概率 7%）
- ……

**曼谷**

- 03.31（暂无预报）
- ……

**04.02 回到杭州（15-22℃，降水概率 30%）**
