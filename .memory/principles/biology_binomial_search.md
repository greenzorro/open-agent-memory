# 生物物种研究标准搜索协议

## 核心原则

研究任何生物物种时，必须先确定其**生物学二名法名称**（Binomial Nomenclature，即拉丁学名），后续所有信息搜索**一律使用二名法名称作为检索词**。

## 为什么必须用二名法

- **过滤伪科普**：中文名在自媒体和商业内容中大量使用，搜索结果掺杂大量不实信息；二名法名称天然过滤掉这类内容
- **消除歧义**：同一中文名可能对应多个物种（如"杜鹃"指代多个属），二名法名称全球唯一
- **信息质量跃升**：以二名法名称命名的网页，来源通常是学术文献、百科全书、自然保护区或博物馆等专业机构
- **跨语言一致性**：二名法是国际通用标准，英文、日文、德文等外文搜索同样适用

## 执行流程

1. **第一步**：用中文名进行初步搜索，仅为了找到该物种的二名法名称
2. **锁定学名**：从维基百科、iNaturalist、GBIF 等可信来源确认学名（格式：`Genus species`，如 `Ailuropoda melanoleuca`）
3. **第二步起**：所有后续搜索完全使用二名法名称，不再使用中文名
4. **搜索协议**：遵循"长尾知识验证协议"（`correction/长尾知识验证协议`），以英文搜索为主

## 示例

| 错误做法 | 正确做法 |
|---------|---------|
| 搜索"水豚" | 先找到学名 `Hydrochoerus hydrochaeris`，再用学名搜索 |
| 搜索"鲸鲨吃什么" | 用学名 `Rhincodon typus` + diet 搜索 |
| 搜索"十大最毒的蛇" | 用具体物种学名搜索毒液成分和毒性数据 |

## 与现有协议的关系

- 本协议是**搜索关键词层面的过滤器**，与 `长尾知识验证协议`（内容层面的验证）互补
- 与 `OSINT 调查认知修正`（跨语言搜索规则）协同：二名法天然解决了跨语言搜索问题
